# i18n/ — backlog

`translator.choice("items", count=5)` and `AcceptLanguageMiddleware`
landed. The choice resolver supports CLDR plural categories
(zero/one/few/many/other) and ships rules for English,
Russian/Ukrainian/Belarusian, Polish, French, and a default-English
fallback. The middleware parses the standard ``Accept-Language``
header (with quality factors) and picks the best supported locale
from the translator's known list, installing it via
:func:`with_locale`.

## YAML and Python catalogues

JSON is fine for translators but verbose. Add YAML and `messages.py`
loaders behind `pylar[i18n-yaml]` so teams can pick the format that
matches their workflow.

## URL-based locale prefix

A routing helper that produces ``/en/...`` / ``/ru/...`` prefix groups
and sets the active locale before dispatching the inner route.

## Date / number formatting

Bind a locale-aware formatter (`Formatter.date(value)`,
`Formatter.number(value)`, `Formatter.currency(amount, currency)`)
backed by `babel`. Optional dep behind `pylar[i18n-babel]`.

## `make:trans` extraction

A console command that scans templates and Python source for
`translator.get(...)` calls and writes the missing keys into the
catalogue files. Mirrors Django's `makemessages`.
